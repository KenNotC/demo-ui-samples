Getting Started

GlobalDocs Atlas

A static HTML documentation system for developers and small teams. No framework. No build step. No dependency to maintain. Open a file, edit content, deploy anywhere.

What this is

GlobalDocs Atlas is a multi-page HTML documentation template built for developers who want professional-looking docs without a React stack. Every page shares the same layout shell — sidebar, topbar, light/dark toggle — and you add content by editing plain HTML.

Zero dependencies
No npm. No build. One HTML file per page.
Light & dark
CSS custom properties. Instant toggle. No flicker.
Multi-page ready
Add a page by duplicating a file. Navigation updates in one place.
Deploy anywhere
GitHub Pages, Netlify, Vercel, or any static host.

Search — powered by Pagefind

GlobalDocs Atlas includes full-text search powered by Pagefind — a fast, static search engine that runs entirely in the browser with no server required.

The search bar is already integrated in the topbar. To activate it after editing your content, run the indexer once:

shell
npx pagefind --site .

This scans your HTML files and generates the search index inside a pagefind/ folder. Run it again whenever you publish new content. See the Quick Start for full instructions.

Search works on deployed sites (GitHub Pages, Netlify, Vercel). It does not work when opening files directly from your desktop as file:// URLs — serve locally with npx serve . to test search before deploying.

How navigation works

The sidebar is the same across every page. When you add a new page, add its link to the sidebar. Mark the current page's link as class="nav-link active" — that's all the active state needs.

The sidebar navigation is plain HTML — no JavaScript required to render it. Update it in one template and copy it to each page.

What you need to know

To use this template you need to be comfortable with:

  • Editing HTML files in a text editor
  • Uploading files to a static host

You do not need to know CSS, JavaScript, or any framework.

This template does not have a CMS or auto-generated navigation — these are intentional omissions, not missing features. Search is included and requires one command to activate. If you need a full CMS, look at Docusaurus or GitBook instead.
