Guide

Site Structure

How GlobalDocs Atlas organises pages, navigation, content, and search — and how to extend it.

How pages work

Each page is a self-contained HTML file. No server, no build process, no shared state. The browser loads each file directly. Every page has three sections: Topbar (brand, search, theme toggle), Sidebar (navigation, same across all pages), and Content area (unique per page).

The pagefind/ folder is generated when you run npx pagefind --site .. It does not exist on first download. Include it when deploying.

Adding a new page

  1. Duplicate any existing .html file
  2. Rename it (e.g. guide-billing.html)
  3. Change the <title> tag
  4. Replace content inside <div class="content">
  5. Add a link in the sidebar of every page
  6. Set class="nav-link active" only in the new file
  7. Update breadcrumb and prev/next buttons
  8. Run npx pagefind --site . to index the new page
